Robot Simulation Project: Key Portfolio Points

The following points summarize the essential technical and methodological achievements of the individual Robot Simulation Assignment (RME40003) using ABB RobotStudio®. These highlight skills in offline programming, path planning, and industrial robot operation.

1. Project Objective and Core Tools

  • Primary Objective: To emulate the functions of a robot using **ABB RobotStudio®** and guide a robotic arm to accurately trace a model design or symbol (the "RSD" flag) within a defined workspace (on a table).
  • Software Utilization: Demonstrated proficiency in **ABB RobotStudio®** (utilizing features like **offline programming**) and **SolidWorks** for initial model design.
  • Model Transfer: Successfully designed the custom **"RSD" flag model** in SolidWorks and managed the file transfer process by exporting the model as a **.sat file** and importing it into RobotStudio using the 'Import Geometry' feature.
  • Workspace Definition: The imported model was accurately positioned, aligned, and set within the **User Coordinate System (UCS)**, which served as the default reference frame for the robot’s operations.

2. Technical Specifications of the Robot

The project utilized the ABB IRB 1600 model, showcasing understanding of industrial robot capabilities:

  • Robot Model: **ABB IRB 1600**.
  • Axes and Versatility: The IRB 1600 is a versatile robot model with **six axes**.
  • Reach and Payload: Features a **1.2 meters reach** and a load capacity (payload) of **6kg**.
  • Precision: High accuracy is evidenced by its **repeatability of 0.02**.
  • End Effector: The simulation used the **"MyTool"** end effector, which functionally acts as a pen for the tracing task.

3. Advanced Path Planning and Motion Control

Demonstrated complex programming skills beyond simple linear movement:

  • Handling Complex Features: For letters with **curvy features** (like 'R' and 'S'), linear paths were merged to implement the **‘MoveC’ command** (circular path motion). This was crucial for ensuring the **smoothness and continuity** of the tracing, avoiding sharp mechanical transitions.
  • Programming Precision: While using the Auto Path feature, some paths were also created **manually** to improve the **precision and detail** of the pen’s movement.
  • Z-Coordinate Manipulation: Successfully programmed transitions between figures by creating new targets and paths and **modifying the z-coordinate**. This allowed the tool to **lift off the surface** after tracing one figure and accurately descend to start the next one.

4. RAPID Code Implementation and Simulation Management

  • Structured Code Execution: The four initial paths were merged into a single path named **COMPLETE PATH**. The **RAPID code** was structured to call this path in the main procedure function (`PROC main() COMPLETE_PATH; MOVE_HOME; ENDPROC`), ensuring the entire tracing task executed as a single function call.
  • Simulation Control: A `MOVE_HOME` function was implemented, along with a **WaitTime 20** function, allowing the simulation to pause before restarting the task. This enabled the robot to run in **continuous run mode** for recording.
  • Deliverables: The project successfully achieved the simulation and tracing goal, with deliverables including a video demonstration and the **Pack and Go file** from RobotStudio, which allows the simulation programme to be accessed on other computers.
